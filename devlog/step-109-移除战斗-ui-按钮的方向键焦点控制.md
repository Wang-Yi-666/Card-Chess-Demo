# Step 109 - 移除战斗 UI 按钮的方向键焦点控制

## 日期

2026-04-02

## 问题

战斗中的 UI 按钮会响应方向键切换焦点，这不符合接下来正式化战斗 UI 的需求，也容易和战斗本身的输入逻辑混在一起。

排查后确认：

- 战斗逻辑层没有专门写“方向键切按钮”的输入分支
- 实际来源是 Godot `Button` 的默认键盘焦点导航

## 本次改动

更新文件：

- `Scripts/Battle/UI/BattleHudController.cs`

处理方式：

- 为战斗 HUD 的按钮统一关闭键盘焦点
- 将主要操作按钮的 `FocusMode` 设为 `None`
- 同时补上动作记录弹窗、牌堆弹窗的关闭按钮与遮罩按钮，避免这些按钮继续参与方向键导航

## 结果

现在战斗中的这些 UI 按钮不会再通过方向键切换焦点：

- 右侧操作按钮
- 荒川按钮与荒川轮盘按钮
- 动作记录弹窗关闭按钮
- 牌堆弹窗关闭按钮

这次改动只影响战斗 UI 的键盘焦点行为，不改战斗逻辑和地图 / 战斗接口。
