# Step 114 - 把战斗背景改为插入房间内部显示层

## 日期

2026-04-02

## 问题

上一轮通过降低 `FloorLayer` 透明度来透出背景图，但实际效果只是让棋盘发灰，背景图仍然没有真正显示。

这说明“背景在房间底图后面，再靠透明度透出来”这条路不稳。

## 本次调整

更新文件：

- `Scripts/Battle/BattleSceneController.cs`

改动思路：

- 不再让背景停留在 `RoomContainer` 底层
- 不再依赖把 `FloorLayer` 调半透明来透出背景
- 改为在运行时把 `BattleBackground` 直接重挂到 `CurrentRoom` 里
- 插入位置放到：
  - `FloorLayer` 之后
  - 单位、覆盖层、浮字之前

这样背景依旧属于战斗世界层，但不会继续被房间底图完全盖住。

## 同时撤销

撤销了上一轮的地板半透明做法：

- 删除 `BattleFloorAlpha`
- 删除运行时对 `FloorLayer.Modulate.a` 的修改

这样棋盘不会再继续整体发灰。

## 结果

现在战斗背景的层级关系应当是：

- 地板层
- 背景图
- 单位 / 棋盘覆盖 / 浮字

这保证了：

- 背景不是 UI
- 会跟随棋盘和镜头移动
- 不做视差
- 也不会再被底图完全挡住
