# step-148 修正牌堆关闭按钮命中层级并清理牌堆标题乱码

时间：2026-04-05

本次改动目标：
- 修复战斗内牌堆弹窗右上角 `X` 关闭按钮无法高亮、无法点击的问题。
- 清理牌堆弹窗标题和页签被旧乱码逻辑反复覆盖的问题。
- 给关闭按钮补上明确的 hover / pressed 样式，避免看起来像纯文本。

本次实际改动：
- 修改 `Scene/Battle/UI/BattleHud.tscn`
  - 将 `PilePopup/Margin/VBox/Header` 的 `mouse_filter` 从 `Ignore(2)` 调整为 `Pass(1)`。
  - 这样标题栏不再把自己的子节点命中一起吞掉，`CloseButton` 可以正常接收悬停和点击。
- 修改 `Scripts/Battle/UI/BattleHudController.cs`
  - 删除牌堆弹窗对 `GuiInput` 的重复挂接，保留统一的 `_Input()` 全局接管逻辑。
  - 清理 `RefreshPilePopup()` 中多段残留乱码与重复赋值，统一为：
    - 标题：`牌堆查看`
    - 页签：`抽牌堆 / 弃牌堆 / 消耗堆`
  - 新增 `ApplyWindowCloseButtonStyle()`，给右上角 `X` 提供明确的按钮外观与 hover / pressed 状态。
  - 同时把战斗日志弹窗关闭按钮也统一到同一套窗口关闭按钮样式。

问题判断：
- 这次 `X` 无法点击的核心不是“没做成 Button”，它本来就是 `Button`。
- 根因更接近“层级与鼠标过滤配置错误”：
  - 父级 `Header` 设成 `Ignore` 后，关闭按钮无法稳定收到悬停/点击。
  - 同时按钮本身之前没有明显的窗口按钮样式，所以视觉上也像一段普通文字。

当前约束：
- 仍然坚持不改战斗与地图接口规范，只修 HUD 本侧实现。
- 牌堆弹窗的外部关闭仍由 `_Input()` 统一处理，避免后续和底层战斗按钮再发生竞争。

后续验证点：
1. 鼠标移到 `X` 上是否会高亮。
2. 点击 `X` 是否立即关闭牌堆弹窗。
3. 点击弹窗外空白区域是否也能关闭。
