# 标准内容示范包

本目录用于演示如何基于现有模板，实际组织一套可用的：

- 卡牌内容
- 天赋内容
- 装备内容
- 敌人招牌技能与学习目标
- 精英 / Boss 奖励
- starter deck 与升级路线

这不是最终正式数据，而是一套“结构正确、平衡方向清晰、足够拿来继续扩写”的示范。

## 1. 本示范包覆盖的设计重点

这套示范内容刻意覆盖了当前项目最重要的几种系统关系：

- 近战 / 远程 / 灵活 三条基础分支
- 学习机制入口卡
- 学习获得的精英 / Boss 特色卡
- 超规携带的适用对象
- Quick 与循环牌的平衡约束
- 影响因子作为牌组负载控制器
- 装备对构筑和学习机制的轻量修正

## 2. 文件说明

- `卡牌设计示范.csv`
  - 提供一组标准示范卡牌
- `天赋设计示范.csv`
  - 提供与卡牌配套的成长树示范节点
- `装备设计示范.csv`
  - 提供影响构筑与学习机制的装备示范
- `敌人招牌技能与学习目标示范.csv`
  - 提供敌人招牌技能、学习条件与对应奖励牌示范
- `精英与Boss奖励示范.csv`
  - 提供战斗奖励结构示范
- `starter_deck与升级路线示范.csv`
  - 提供从起始牌组到中期 build 的示范路线

## 3. 这套示范内容的大致玩法轮廓

### 近战

- 依靠基础斩击与稳守建立站位压迫
- 中期通过防御强化和高压突进形成贴身爆发
- 学习牌偏高风险高收益

### 远程

- 依靠直线攻击和距离控制稳定输出
- 中期转向压制与高值远程牌
- 学习牌偏单卡强度高，复制上限严格

### 灵活

- 依靠 Quick、过牌、机动与低费调度运转
- 但所有危险循环牌都被强制压上 `Exhaust`、同名限制或较高影响因子
- 学习机制入口也挂在灵活 / 通用路线，强调“观察与适配”

## 4. 推荐如何阅读这套示范

推荐顺序：

1. 先看 `天赋设计示范.csv`
2. 再看 `卡牌设计示范.csv`
3. 再看 `敌人招牌技能与学习目标示范.csv`
4. 再看 `精英与Boss奖励示范.csv`
5. 最后看 `starter_deck与升级路线示范.csv`

原因是：

- 天赋决定长期成长方向
- 卡牌负责把方向变成具体战斗语言
- 敌人学习表决定高价值卡从哪里来
- 奖励表示范战斗外反馈节奏
- 升级路线表把这些内容串成真实成长路径

## 5. 这套示范刻意做了哪些平衡处理

### 1. 没有固定最高牌数

牌组厚度由：

- 最低牌数要求
- 影响因子总上限

共同决定。

### 2. 危险牌默认不让堆

例如：

- 0 费回能
- Quick 抽牌
- Quick 回能

都通过以下方式限制：

- `Exhaust`
- 同名上限 `1`
- 影响因子偏高

### 3. 学习牌不默认直接适配任何流派

学习获得的牌虽然会进入“已拥有”，但是否能正常带入仍看成长要求。

### 4. 装备只能补短板，不能跳过整条成长线

装备主要做：

- 分支专精临时 +1
- 影响因子承载小幅提高
- 学习行为修正

不会直接让玩家无条件绕过全部限制。

## 6. 下一步如何继续扩写

你可以直接基于这套示范做两种工作：

### 方式 A：横向扩卡

保持原分支结构不动，继续补：

- 同分支不同稀有度
- 同分支不同费用段
- 同分支不同目标类型

### 方式 B：纵向扩成长

保持卡牌数量不大幅增加，继续补：

- 更深层分支节点
- 学习机制强化节点
- 影响因子承载型角色能力节点
