# 90度方格战棋需求表

## 1. 文档定位

本表按当前仓库实际状态整理“战斗原型需求完成度”。

状态含义：

- `已实现`
- `部分实现`
- `未实现`

| ID | 模块 | 当前状态 | 说明 |

## 3. 当前已实现的战斗原型能力

### 3.1 棋盘与对象

- 16x8 默认棋盘
- `BoardObject` 统一对象模型
- registry、cell state、occupancy rules 已接通
- 房间模板可通过 marker layer 生成对象出生点

### 3.2 运行时表现

- 玩家、敌人、障碍物 prefab 已接通
- 对象状态可同步到 HUD
- prefab 支持基础动画名切换

### 3.3 回合测试

- 玩家移动力可以实时调试
- 每回合只能移动一次
- 可显式结束回合
- 可进入下一回合

## 4. 当前未实现但常被误写成已实现的内容

以下需求仍然只是后续目标：

- 普通攻击
- 防御
- 冥想
- 卡牌
- 能量
- targeting
- 时间推进
- 地图到战斗的数据请求

## 5. 当前验收用例

当前版本实际可用的验收用例应当是：

1. 启动项目后进入战斗测试场景。
2. 棋盘、玩家、敌人、障碍物正确显示。
3. HUD 正确显示玩家 HP、移动力、回合状态。
4. 左键可以在合法目标格内移动玩家。
5. 同一回合内第二次移动会被禁止。
6. 点击 `End` 或按 `T / Enter` 后结束当前回合。
7. 点击 `Next` 或再次按 `T / Enter` 后进入下一回合。
8. 点击 `MV- / MV+` 后移动力和可达区域同步变化。
