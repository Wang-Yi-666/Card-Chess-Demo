- `已实现`
- `部分实现`

## 3. 当前已实现的战斗原型能力

### 3.1 棋盘与对象

- 16x8 默认棋盘
- `BoardObject` 统一对象模型
- registry、cell state、occupancy rules 已接通
- `BoardTopology` 统一管理四方向格子索引
- 房间模板可通过 marker layer 生成对象出生点

### 3.2 运行时表现

- 玩家、敌人、障碍物 prefab 已接通
- 对象状态可同步到 HUD
- prefab 支持基础动画名切换
- 障碍物已分出可破坏 / 墙体 / 慢速可通过三类原型

### 3.3 工具层

- 已有 `BoardPathfinder`，支持 A* 路径查询
- 已有 `BoardTargetingService`，支持四方向直线索敌查询
- 当前索敌工具已接入直线卡牌，但还未接入敌人 AI
- 当前伤害链路已接入护盾先于生命的结算顺序

### 3.4 回合测试

- 玩家移动力可以实时调试
- 每回合只能移动一次
- 玩家可以跳过移动
- 可显式结束回合
- 已有最小手牌、能量和弃牌堆循环
- 已有荒川全局能量与顶部 HUD 显示
- 荒川轮盘只在玩家活动阶段可用，且不消耗玩家移动/行动状态
- `Quick` 卡牌可以在不结束回合的情况下结算
- 已有冥想按钮与护盾卡测试样本
- 普通攻击后会自动进入回合后处理
- 敌方已接入最小 AI 回合执行

## 4. 当前未实现但常被误写成已实现的内容

以下需求仍然只是后续目标：

- 完整冥想系统
- 完整卡牌内容与关键词
- 完整能量解耦
- 完整 targeting 规则接入
- 时间推进
- 地图到战斗的数据请求

## 5. 当前验收用例

当前版本实际可用的验收用例应当是：

1. 启动项目后进入战斗测试场景。
2. 棋盘、玩家、敌人、障碍物正确显示。
3. HUD 正确显示玩家 HP、移动力、回合状态。
4. 左键可以在合法目标格内移动玩家。
5. 同一回合内第二次移动会被禁止。
6. 点击 `End` 或按 `T / Enter` 后结束当前回合。
7. 点击 `Next` 或再次按 `T / Enter` 后进入下一回合。
8. 点击 `MV- / MV+` 后移动力和可达区域同步变化。
