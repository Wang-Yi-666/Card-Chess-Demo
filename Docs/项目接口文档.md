# 项目接口文档

## 1. 文档目标

本文件用于定义整个项目中地图层、战斗层、全局状态层之间的标准接口。

目标是：

- 让地图探索和棋盘战斗可以并行开发。
- 保证跨场景数据只有一套事实来源。
- 限制模块之间的直连，避免后期耦合失控。
- 为后续扩展地图事件、更多战斗系统、局外成长、存档、剧情分支预留稳定接口。

## 2. 接口设计原则

### 2.1 单一事实源

所有跨地图、跨战斗、跨场景的持久状态，统一以 `GameSession` 为唯一事实源。

### 2.2 单向数据流

推荐的数据流为：

`地图层 -> BattleRequest -> 战斗层 -> BattleResult -> GameFlowController -> 地图层 / GameSession`

### 2.3 单写者规则

每一类数据只允许一个主要写入方：

- 地图状态主要由地图层写入。
- 战斗结算结果主要由战斗层写入。
- 持久状态主要由 `GameSession` 写入。
- 场景切换只允许 `GameFlowController` 触发。

### 2.4 面向数据而不是节点

接口传递必须优先使用纯数据对象，而不是传场景节点引用。

禁止做法：

- 地图层直接拿战斗场景节点调用方法。
- 战斗层直接改地图场景中的敌人节点。
- 用全局变量零散同步玩家 HP、牌组、燃料。

## 3. 模块关系

```text
WorldMap
  -> BattleRequest
  -> GameFlowController.start_battle(request)
  -> BattleScene
  -> BattleResult
  -> GameFlowController.finish_battle(result)
  -> GameSession.apply_battle_result(result)
  -> WorldMap.resume_from_context(context)
```

## 4. 关键接口总览

| 接口名 | 作用 | 主要写入方 | 主要读取方 |
| --- | --- | --- | --- |
| `GameSession` | 全局持久状态容器 | GameFlowController / Meta系统 | 地图层、战斗层、UI |
| `RuntimeSnapshot` | 进入战斗时的运行时快照 | GameSession | BattleRequest |
| `BattleRequest` | 地图进入战斗时的标准输入 | 地图层 | 战斗层 |
| `BattleResult` | 战斗结束时的标准输出 | 战斗层 | GameFlowController / 地图层 |
| `MapResumeContext` | 返回地图时的恢复参数 | GameFlowController | 地图层 |
| `InventoryDelta` | 物品变化结果 | 地图层 / 战斗层 | GameSession |
| `ResourceDelta` | 资源变化结果 | 地图层 / 战斗层 | GameSession |
| `FlagDelta` | 剧情与系统 Flag 变化 | 地图层 / 战斗层 | GameSession |

## 5. GameSession

### 5.1 作用

`GameSession` 是整个项目的全局持久状态容器，负责保存跨场景状态。

### 5.2 推荐字段

| 字段 | 类型 | 必填 | 写入方 | 说明 |
| --- | --- | --- | --- | --- |
| `session_id` | `String` | 是 | 系统 | 本轮游戏唯一 ID |
| `current_map_id` | `StringName` | 是 | 地图层 / Flow | 当前地图标识 |
| `current_map_spawn_id` | `StringName` | 否 | Flow | 返回地图时的出生点 |
| `player_profile_id` | `StringName` | 是 | 系统 | 玩家角色模板 ID |
| `player_runtime` | `Dictionary` 或对象 | 是 | GameSession | 玩家当前 HP、属性、状态 |
| `deck_state` | `Dictionary` 或对象 | 是 | GameSession | 当前牌组、手牌外状态、构筑结果 |
| `inventory_state` | `Dictionary` 或对象 | 是 | GameSession | 持有道具、关键物品 |
| `arakawa_state` | `Dictionary` 或对象 | 是 | GameSession | 荒川资源、等级、能力解锁 |
| `suitcase_state` | `Dictionary` 或对象 | 是 | GameSession | 手提箱燃料、过载状态、模组 |
| `world_flags` | `Dictionary[StringName, Variant]` | 是 | GameSession | 全局剧情与系统 Flag |
| `scan_risk` | `int` | 否 | GameSession | 当前黑户 / 非法 AI 暴露风险 |
| `cleared_encounters` | `Array[StringName]` | 否 | GameSession | 已清除的地图遭遇 |
| `used_interactables` | `Array[StringName]` | 否 | GameSession | 已使用补给站 / 交互点 |

### 5.3 推荐约束

- `GameSession` 不依赖任何场景节点。
- 所有字段必须可序列化，便于存档。
- 不在 `GameSession` 中保存 Node 引用。

## 6. RuntimeSnapshot

### 6.1 作用

`RuntimeSnapshot` 表示进入战斗前的一次状态快照，用于让战斗层在初始化时获得稳定输入。

### 6.2 推荐字段

| 字段 | 类型 | 必填 | 来源 | 说明 |
| --- | --- | --- | --- | --- |
| `snapshot_id` | `String` | 是 | 系统 | 快照 ID |
| `player_hp` | `int` | 是 | GameSession | 当前生命值 |
| `player_stat_modifiers` | `Dictionary` | 否 | GameSession | 地图事件带来的临时修正 |
| `deck_runtime_seed` | `int` | 否 | GameSession | 洗牌或生成随机种子 |
| `arakawa_energy` | `int` | 是 | GameSession | 当前荒川可用资源 |
| `suitcase_fuel` | `int` | 是 | GameSession | 当前手提箱燃料 |
| `active_flags` | `Dictionary` | 否 | GameSession | 战斗初始化要用到的 Flag |

## 7. BattleRequest

### 7.1 作用

地图层进入战斗时，必须只组装一个 `BattleRequest` 并交给 `GameFlowController`。

### 7.2 推荐字段

| 字段 | 类型 | 必填 | 写入方 | 说明 |
| --- | --- | --- | --- | --- |
| `request_id` | `String` | 是 | 地图层 | 请求唯一 ID |
| `encounter_id` | `StringName` | 是 | 地图层 | 遭遇定义 ID |
| `map_id` | `StringName` | 是 | 地图层 | 来源地图 |
| `map_node_id` | `StringName` | 否 | 地图层 | 来源地图节点 |
| `encounter_trigger_id` | `StringName` | 否 | 地图层 | 触发器 ID |
| `battlefield_theme_id` | `StringName` | 是 | 地图层 / 内容表 | 战场视觉主题 |
| `room_rule_tags` | `Array[StringName]` | 否 | 地图层 | 如 `pirate`, `scan_zone`, `boss_room` |
| `enemy_group_id` | `StringName` | 是 | 地图层 / 内容表 | 敌人组配置 |
| `reward_table_id` | `StringName` | 否 | 地图层 | 掉落规则 |
| `player_snapshot` | `Dictionary` 或对象 | 是 | GameSession | 玩家运行时快照 |
| `deck_snapshot` | `Dictionary` 或对象 | 是 | GameSession | 牌组快照 |
| `arakawa_snapshot` | `Dictionary` 或对象 | 是 | GameSession | 荒川状态快照 |
| `suitcase_snapshot` | `Dictionary` 或对象 | 是 | GameSession | 手提箱状态快照 |
| `global_flags` | `Dictionary` | 否 | GameSession | 全局 Flag 快照 |
| `random_seed` | `int` | 否 | Flow | 本次战斗种子 |

### 7.3 参数说明建议

- `encounter_id` 用来决定战斗内容逻辑，不直接决定场景文件路径。
- `battlefield_theme_id` 只影响表现主题和部分规则标签，不应承载所有配置。
- `room_rule_tags` 用于可扩展筛选，如：
  - `tight_corridor`
  - `fuel_hazard`
  - `scan_pressure`
  - `arakawa_boosted`
- `random_seed` 必须在调试模式可重放。

## 8. BattleResult

### 8.1 作用

战斗结束后，战斗层必须只输出一个 `BattleResult`，由 `GameFlowController` 和 `GameSession` 统一消费。

### 8.2 推荐字段

| 字段 | 类型 | 必填 | 写入方 | 说明 |
| --- | --- | --- | --- | --- |
| `result_id` | `String` | 是 | 战斗层 | 结果唯一 ID |
| `request_id` | `String` | 是 | 战斗层 | 对应的 BattleRequest |
| `victory` | `bool` | 是 | 战斗层 | 是否胜利 |
| `defeat_reason` | `StringName` | 否 | 战斗层 | 如 `hp_zero`, `mission_fail` |
| `remaining_hp` | `int` | 是 | 战斗层 | 战后剩余 HP |
| `hp_delta` | `int` | 是 | 战斗层 | 战前战后差值 |
| `status_delta` | `Array[Dictionary]` | 否 | 战斗层 | 状态增加/移除 |
| `deck_delta` | `Dictionary` | 否 | 战斗层 | 牌组、临时牌、消耗变化 |
| `arakawa_energy_delta` | `int` | 是 | 战斗层 | 荒川资源变化 |
| `fuel_delta` | `int` | 是 | 战斗层 | 燃料变化 |
| `inventory_delta` | `Array[Dictionary]` | 否 | 战斗层 | 掉落或消耗物品 |
| `new_flags` | `Dictionary[StringName, Variant]` | 否 | 战斗层 | 新增 Flag |
| `cleared_encounter_id` | `StringName` | 否 | 战斗层 | 清除的遭遇 |
| `triggered_story_ids` | `Array[StringName]` | 否 | 战斗层 | 触发剧情 |
| `next_map_spawn_id` | `StringName` | 否 | 战斗层 / Flow | 返回地图落点 |

### 8.3 结果消费规则

- `GameSession` 负责把数值变化落盘。
- 地图层负责根据 `cleared_encounter_id` 清除地图敌人。
- 地图层负责根据 `triggered_story_ids` 触发对话或事件。
- 地图层不应重新计算战斗结果。

## 9. MapResumeContext

### 9.1 作用

`MapResumeContext` 用于战斗结束返回地图时恢复地图状态。

### 9.2 推荐字段

| 字段 | 类型 | 必填 | 写入方 | 说明 |
| --- | --- | --- | --- | --- |
| `map_id` | `StringName` | 是 | Flow | 返回地图 ID |
| `spawn_id` | `StringName` | 否 | Flow | 回到地图的位置锚点 |
| `focus_node_id` | `StringName` | 否 | Flow | 镜头或 UI 聚焦节点 |
| `cleared_encounter_id` | `StringName` | 否 | BattleResult | 需要移除的遭遇 |
| `updated_flags` | `Dictionary` | 否 | GameSession | 地图恢复时需应用的全局 Flag |
| `consumed_interactable_ids` | `Array[StringName]` | 否 | GameSession | 已消耗的交互点 |

## 10. InventoryDelta

### 10.1 作用

统一描述物品变化，避免地图和战斗各写一套格式。

### 10.2 推荐字段

| 字段 | 类型 | 必填 | 说明 |
| --- | --- | --- | --- |
| `item_id` | `StringName` | 是 | 物品定义 ID |
| `amount` | `int` | 是 | 正数为获得，负数为消耗 |
| `reason` | `StringName` | 否 | 如 `pickup`, `battle_reward`, `consume` |
| `source_id` | `StringName` | 否 | 来源对象 ID |

## 11. ResourceDelta

### 11.1 作用

统一描述资源变化，如荒川资源、燃料、扫描风险。

### 11.2 推荐字段

| 字段 | 类型 | 必填 | 说明 |
| --- | --- | --- | --- |
| `resource_key` | `StringName` | 是 | 如 `arakawa_energy`, `suitcase_fuel`, `scan_risk` |
| `delta` | `int` | 是 | 变化量 |
| `reason` | `StringName` | 否 | 变化原因 |
| `clamp_min` | `int` | 否 | 最小值 |
| `clamp_max` | `int` | 否 | 最大值 |

## 12. FlagDelta

### 12.1 作用

统一描述剧情和系统 Flag 变化。

### 12.2 推荐字段

| 字段 | 类型 | 必填 | 说明 |
| --- | --- | --- | --- |
| `flag_key` | `StringName` | 是 | Flag 名称 |
| `value` | `Variant` | 是 | 新值 |
| `reason` | `StringName` | 否 | 触发原因 |

## 13. GameFlowController 接口

### 13.1 推荐公开方法

```gdscript
func start_battle(request: BattleRequest) -> void
func finish_battle(result: BattleResult) -> void
func return_to_map(context: MapResumeContext) -> void
func save_game(slot_id: StringName) -> void
func load_game(slot_id: StringName) -> void
```

### 13.2 职责

- 校验 `BattleRequest`
- 切换场景
- 持有当前流程状态
- 接收 `BattleResult`
- 更新 `GameSession`
- 生成 `MapResumeContext`

## 14. 地图层接口建议

### 14.1 推荐公开方法

```gdscript
func build_battle_request(encounter_trigger_id: StringName) -> BattleRequest
func resume_from_context(context: MapResumeContext) -> void
func apply_session_state(session: GameSession) -> void
```

### 14.2 地图层禁止事项

- 不直接实例化战斗核心逻辑对象作为自己的子节点长期持有。
- 不直接写 `BattleResult`。
- 不直接维护战斗中的牌组状态。

## 15. 战斗层接口建议

### 15.1 推荐公开方法

```gdscript
func initialize_from_request(request: BattleRequest) -> void
func run_battle() -> void
func build_battle_result() -> BattleResult
```

### 15.2 战斗层禁止事项

- 不直接调用地图场景节点的方法。
- 不直接操作补给站或地图拾取物节点。
- 不直接保存持久存档。

## 16. 参数命名规范建议

### 16.1 ID 类字段

统一使用：

- `*_id` 表示定义表或运行时对象 ID
- `*_ids` 表示 ID 数组

示例：

- `map_id`
- `encounter_id`
- `enemy_group_id`
- `triggered_story_ids`

### 16.2 快照类字段

统一使用：

- `*_snapshot`

示例：

- `player_snapshot`
- `deck_snapshot`
- `suitcase_snapshot`

### 16.3 差值类字段

统一使用：

- `*_delta`

示例：

- `hp_delta`
- `fuel_delta`
- `arakawa_energy_delta`

### 16.4 Flag 命名建议

统一使用小写 snake_case 或 `StringName`：

- `met_arakawa`
- `passed_starport_scan`
- `cleared_hitchhiker_ship`
- `hunter_alert_level`

## 17. 可扩展性预留

这套接口应支持后续扩展而不破坏旧逻辑：

- 新地图事件类型
- 新战斗模式
- 新资源类型
- 新剧情 Flag
- 新掉落规则
- 新局外成长系统

建议做法：

- 允许 `Dictionary` 中追加字段，但旧字段语义不变。
- 尽量通过 `tags` 和 `flags` 扩展，而不是频繁改主结构。
- 对外文档优先标注“必填字段”和“可选字段”。

## 18. 最小落地顺序

1. 先定义 `BattleRequest`、`BattleResult`、`MapResumeContext`。
2. 再定义 `GameSession` 的最小字段。
3. 然后打通 `GameFlowController.start_battle()` 和 `finish_battle()`。
4. 最后再逐步补 `InventoryDelta`、`ResourceDelta`、`FlagDelta` 等细分对象。

## 19. 结论

如果你和你朋友都按这份接口文档开发，那么地图层和战斗层可以真正解耦。

你们只需要对齐：

- 进入战斗时传什么
- 战斗结束时回什么
- 全局状态由谁维护
- 哪些字段可以扩展

这样后面无论是加地图事件、Boss 房、扫描系统、更多卡牌，还是做存档和回放，都不用推翻现有结构。
