# 项目接口文档

## 1. 文档目标

本文件定义地图层、战斗层、全局状态层之间的标准接口，同时补充战斗层内部最关键的数据对象。

重点新增：

- `BoardObject`
- `BoardCellState`
- `RoomLayoutDefinition`
- `TurnActionState`
- 能量、卡牌、索敌相关接口

## 1.1 命名规范统一

为避免不同文档对同一功能出现多套命名，本项目文档统一采用以下规则：

- 运行时类型、服务、控制器、命令、资源类型使用 `PascalCase`
- 方法名使用 `PascalCase`
- 目录与文件名跟当前 C# 工程保持一致，使用 `Scripts/Battle/.../*.cs`
- 仅在表达内容层字段、存档键名或表字段语义时保留 `snake_case`

例如：

- 类型：`BoardInitializer`
- 文件：`Scripts/Battle/Board/BoardInitializer.cs`
- 方法：`InitializeFromLayout(...)`
- 数据字段：`layout_id`

## 2. 顶层数据流

`地图层 -> BattleRequest -> 战斗层 -> BattleResult -> GameFlowController -> GameSession / 地图层`

## 3. 顶层接口

| 接口名 | 作用 | 写入方 | 读取方 |
| --- | --- | --- | --- |
| `GameSession` | 全局持久状态 | Flow / Meta | 地图层、战斗层 |
| `BattleRequest` | 进入战斗输入 | 地图层 | 战斗层 |
| `BattleResult` | 战斗结束输出 | 战斗层 | Flow / 地图层 |
| `MapResumeContext` | 返回地图恢复参数 | Flow | 地图层 |

## 18. 关键规则约束

- 地图层不能直接创建 BoardObject。
- 战斗层不能直接修改地图节点。
- 一切跨场景状态只能回写到 GameSession。
- `BattleResult` 是战斗层唯一对外结果出口。
- 目标获取必须走 TargetingService，不允许卡牌自己遍历场景树找目标。

## 19. 扩展建议

这套接口应允许未来扩展：

- 新 object 类型
- 新卡牌关键词
- 新房间布局规则
- 新资源类型
- 新 targeting 规则

做法：

- 固定核心字段语义
- 新能力尽量通过 `tags`, `payload`, `effect_bundle` 扩展
- 不轻易修改已有 ID 字段语义
