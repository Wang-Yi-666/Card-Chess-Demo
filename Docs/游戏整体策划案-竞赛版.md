# 游戏整体策划案（竞赛版）

## 1. 文档定位

本文档基于当前项目已完成内容、世界观文稿、玩法分析文档和竞赛限制条件，整理一份适合当前 demo 体量的整体策划案。

它不是长期商业版本的完整 GDD，而是面向本次极限开发竞赛的收束版方案。

目标是：

- 用最短的流程展示最强的战斗核心
- 用最少的地图内容交代世界观和角色关系
- 用一个小而完整的切片，让玩家感受到成长、构筑和策略上限

## 2. 当前版本目标

当前 demo 不追求：

- 大地图
- 多章节长流程
- 完整开放式探索
- 大体量剧情分支

当前 demo 要追求的是：

- 一个清晰开场
- 一个明确的系统教学段
- 一个可探索的小型空港区域
- 一个迷宫式战斗切片
- 一个 Boss 战收束

换句话说，整个作品要像一个“高质量垂直切片”，而不是“未完成的长流程”。

## 3. 一句话版本定位

一名失忆的星际牛仔，在流窜 AI 荒川的协助下，从报废飞船中死里逃生，迫降空港，在短暂探索与一段精心设计的迷宫战斗中，逐步掌握卡牌驱动的战棋战斗，最终以一场 Boss 战展示整个系统的策略深度。

## 4. 当前竞赛版推荐流程

## 4.1 整体结构

建议整体流程为：

```text
序章苏醒
-> 幽灵船教程段
-> 遇见荒川
-> 紧急迫降空港
-> 空港小型探索区
-> 进入迷宫
-> 迷宫中的战斗与成长循环
-> 迷宫终点 Boss 战
-> 章节结束
```

## 4.2 时长目标

建议目标时长：

- 首次完整体验：`25 ~ 45` 分钟

这对竞赛版本更合理，因为：

- 足够短，评委能完整体验
- 足够长，系统价值能真正展开

## 5. 详细玩法规划

## 5.1 序章：余烬中苏醒

### 目标

- 给世界观一个强烈第一印象
- 让玩家认出主角和手提箱
- 让“失忆 + 牛仔 + 废船 + 异常科技”的气质先成立

### 内容建议

- 玩家在残骸中醒来
- 可简单移动
- 读两三段极短文本
- 看到飞船正在崩溃

这段不需要复杂玩法，主要作用是建立氛围。

## 5.2 幽灵船教程桥段

### 目标

- 教最基础的地图移动
- 教交互
- 教第一次遇敌
- 教最基础战斗动作

### 推荐教学顺序

1. 地图移动
2. 交互键
3. 遇敌进入战斗
4. 战斗中的：
   - 移动
   - 普通攻击
   - 结束回合
5. 第二场战斗再教：
   - 防御
   - 冥想 / 抽牌
6. 第三场战斗或压轴事件教：
   - 荒川能力

### 为什么教程段要放在飞船里

因为飞船崩溃自带紧迫感，非常适合把教学和剧情动机绑定：

- 玩家不是为了学系统而学系统
- 玩家是在“为了活下来”自然学会系统

## 5.3 遇见荒川

### 目标

- 让世界观中最有魅力的异常点登场
- 给战斗和成长系统一个未来锚点

### 呈现要求

- 荒川登场必须有明显的视觉和叙事冲击
- 不需要讲太多设定，但要让玩家立刻记住她

### 功能意义

荒川出现后，玩家会理解：

- 这个游戏不只是一个边境求生故事
- 还有“异常 AI + 身份谜团 + 世界秩序”这条更大的线

## 5.4 紧急迫降空港

### 目标

- 从封闭教程场景切换到稍微开放的探索区域
- 给玩家一个“节奏缓冲区”
- 让地图探索、NPC 互动和战斗外构筑开始有位置

### 空港区域建议

不需要大，但要有最少的“生活感”和“选择感”：

- 1 个修理 / 补给点
- 1 到 2 个 NPC
- 1 个构筑入口
- 1 个通往迷宫的明确入口

### 这个区域的作用

它本质上是玩家第一次感受到：

- 战斗不是一切
- 局外整理和准备也很重要

## 5.5 迷宫切片

### 目标

这是竞赛版真正的主体内容。

迷宫要承担三件事：

1. 让玩家连续体验战斗
2. 让构筑和成长产生短周期反馈
3. 让房间地形池的价值真正体现出来

### 为什么适合做成 DRPG 式迷宫

因为它非常适合当前项目体量：

- 容易控制内容规模
- 容易设计遭遇节奏
- 容易突出“房间池”和“战术题面变化”
- 不需要做开放世界式内容量

### 迷宫内容建议

建议规模：

- 3 到 5 个主要战斗节点
- 1 到 2 个事件 / 休整节点
- 1 个小精英节点
- 1 个 Boss 终点

### 迷宫体验节奏

推荐循环：

```text
探索一小段
-> 进入一场战斗
-> 获得少量成长 / 资源 / 构筑反馈
-> 继续探索
-> 再战斗
-> 最终 Boss
```

## 5.6 Boss 战

### 目标

Boss 战不是只是“更大血量”，而是要在竞赛版中承担“展示这款游戏策略上限”的任务。

### Boss 战应该展示什么

- 站位的重要性
- 地形池的重要性
- 防御与逃跑的价值
- 卡牌构筑的差异
- 荒川能力的战略意义

### Boss 战设计原则

- 不一定复杂，但必须有鲜明战术要求
- 让玩家感到：
  - 普攻不够
  - 只靠一张强牌也不够
  - 必须综合使用移动、卡牌、防御和资源管理

这场战斗打完，玩家应该能感受到：

“这游戏有深度，而且如果继续展开会很好玩。”

## 6. 竞赛版系统取舍建议

## 6.1 这次必须展示的系统

- 地图移动与交互
- 遇敌进入战斗
- 棋盘移动与普通攻击
- 卡牌出牌
- 防御
- 逃跑
- 荒川能力
- 最小局外构筑
- 战斗后反馈

## 6.2 可以只做骨架、不必完全打满的系统

- 存档系统
- 完整掉落系统
- 大型天赋树
- 多角色队伍系统
- 大量敌人种类
- 长流程地图探索

## 6.3 这次最不该分散精力的方向

- 海量地图内容
- 大量文本剧情
- 大规模装备系统
- 过于复杂的战斗关键词

对竞赛来说，少而准比多而散重要得多。

## 7. 成长与构筑在竞赛版中的落地方式

## 7.1 成长系统

竞赛版不需要完整天赋树，但要让玩家感到：

- 自己有成长
- 这种成长会改变卡牌和战斗方式

最小可行方案：

- 经验值
- 少量专精点
- 极少数关键分支节点

## 7.2 构筑系统

竞赛版的构筑重点不是复杂，而是“能看懂、能感受到差异”。

建议：

- 允许玩家在空港区域进入一次构筑界面
- 能看到可选牌库
- 能编辑当前牌组
- 有最少 / 最多卡数和点数预算限制

这样就足够建立：

- 近战偏好
- 远程偏好
- 灵活偏好

的基础体验。

## 8. 房间池与 roguelike 感

这是当前竞赛版非常值得重点强化的设计点。

### 设计价值

通过丰富的房间地形池设计，可以让每一场遇敌战斗都带有 roguelike 式随机感。

注意，这里的随机感不是：

- 抽数值
- 改伤害
- 纯拼脸

而是：

- 地形变了
- 障碍物位置变了
- 路线和掩体关系变了
- 熟悉的敌人也变成新的战术题

### 为什么它适合当前竞赛版

因为：

- 它比做很多敌人更省内容量
- 它比做很多地图更容易控制规模
- 它能显著提高重复游玩的新鲜感

这是当前项目一个非常好的“以小博大”的设计点。

## 9. 竞赛版美术策略

这一部分应完全服从冲刺现实。

### 视觉方向

- 2D 复古 RPG 地图
- 像素 sprite
- 16x16 tilemap
- 参考 GBA 时代阅读性

### 美术优先级

优先完成：

- 主角
- 荒川识别符号
- 基础敌人
- 房间地形
- 卡牌框架
- HUD 统一风格

不要在竞赛版里追求：

- 大量高精度立绘
- 超大地图
- 复杂多阶段动画

## 10. 竞赛版交付目标

如果这次 demo 做对，评委体验结束后应该留下几个明确印象：

1. 这个游戏的战斗很有想法
2. 卡牌和战棋真的结合起来了
3. 房间地形变化让每场战斗都不完全一样
4. 荒川这个角色和系统很有辨识度
5. 如果继续做下去，这会是一款很有潜力的策略作品

## 11. 当前竞赛版最优先的开发任务

按优先级建议：

### P0

- 完成教程桥段
- 完成空港小型探索区
- 完成迷宫切片
- 完成 Boss 战
- 补足基础美术资源

### P1

- 让构筑系统与成长反馈真正进入主循环
- 补充第一批流派差异更明显的卡牌
- 让房间池差异更清晰

### P2

- 继续丰富剧情文本
- 继续扩地图与敌人内容

## 12. 结论

对于这次极限开发竞赛，最合理的方向不是继续追求“完整大作”，而是做出一个：

- 短
- 完整
- 有辨识度
- 能体现系统上限

的高质量切片。

你现在提出的：

- 幽灵船教程
- 荒川登场
- 紧急迫降空港
- 空港探索
- 迷宫切片
- Boss 战收束

这条方案是正确的，而且和当前项目已经搭好的战斗核心与接口框架是匹配的。

如果按这个范围继续推进，项目是有机会在竞赛版本里交出一份完整而有记忆点的作品的。
