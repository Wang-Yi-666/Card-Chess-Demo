# 项目总体需求表

## 1. 文档定位

本文件只记录“当前仓库对应的真实需求状态”。

状态含义：

- `已实现`：当前代码里已经有可运行实现
- `部分实现`：已经有基础结构，但还不能算完整功能
- `未实现`：当前仓库没有对应系统

## 4. 当前可验收基线

当前项目真实可验收的战斗基线是：

- 启动项目直接进入战斗测试场景
- 从房间模板初始化 16x8 棋盘和对象
- 显示玩家、敌人、障碍物 prefab
- 显示底部 HUD 状态条
- 左键执行简化移动测试
- 每回合只能移动一次
- 可显式结束回合并进入下一回合
- 可调试玩家移动力
