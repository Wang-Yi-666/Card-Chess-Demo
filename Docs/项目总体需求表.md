# 项目总体需求表

## 1. 文档定位

本文件只记录“当前仓库对应的真实需求状态”。

状态含义：

- `已实现`：当前代码里已经有可运行实现
- `部分实现`：已经有基础结构，但还不能算完整功能
- `未实现`：当前仓库没有对应系统

## 4. 当前可验收基线

当前项目真实可验收的战斗基线是：

- 启动项目直接进入战斗测试场景
- 从房间模板初始化 16x8 棋盘和对象
- 显示玩家、敌人、障碍物 prefab
- 显示底部 HUD 状态条
- 左键执行简化移动测试
- 手牌与能量可见
- 可通过 HUD 出牌并结算
- 可查看抽牌堆 / 弃牌堆 / 消耗牌堆
- 可在预设房间中测试三类障碍物
- 每回合只能移动一次
- 可显式结束回合并进入下一回合
- 可通过 `Def` 进入最小防御状态
- 顶部 HUD 会显示荒川能量
- 玩家阶段可打开荒川轮盘并使用独立能力
- 敌人在 `EnemyTurn` 会执行最小 AI，并可攻击玩家
- 玩家被击败时会触发最小失败动画并回写失败结果
- 可调试玩家移动力
