# 战斗机制规则书

## 1. 文档定位

本文件描述当前仓库里已经实现的“战斗原型规则”，不是完整成品规则书。

因此本文件只记录现在能在项目里真实验证到的规则。

## 2. 当前棋盘定义

### 2.1 棋盘

当前默认战斗棋盘来自 `BattleRoomTemplate`，默认尺寸为：

- `16 x 8`
- 单格像素尺寸 `16`

房间场景通过 `FloorLayer` 和 `MarkerLayer` 生成逻辑布局。

### 2.2 对象

当前场上对象统一为 `BoardObject`。

当前实际使用的对象类型只有三类：

- 玩家单位
- 敌人单位
- 障碍物

### 2.3 占位规则

当前规则为：

- 一个格子最多一个单位对象
- 非单位对象根据是否阻挡进入 `blocking` 或 `resident`
- 若对象 `stackable_with_unit = false`，单位不能进入该格

## 3. 当前玩家回合规则

### 3.1 当前回合状态

当前版本使用 `TurnActionState` 保存：

- `TurnIndex`
- `Phase`
- `InputMode`
- `HasMoved`
- `HasActed`
- `HasEndedTurn`
- `QuickChainCount`
- `SelectedCardInstanceId`
- `EnergyRechargeTurnInterval`
- `EnergyRechargeTurnProgress`

### 3.2 当前已实现动作

当前只实现了以下测试动作：

- 移动
- 普通攻击
- 防御
- 荒川能力
- 卡牌使用
- 冥想
- 结束当前回合
- 进入下一回合

## 4. 当前移动规则

### 4.1 触发方式

- 鼠标左键点击棋盘格

### 4.2 当前判定逻辑

当前已接入棋盘工具层，但仍不是完整战术规则：

1. 先取主角当前格。
2. `BoardPathfinder` 按四方向和通行消耗计算可达格；如果当前脚下就是慢速可通过障碍格，则会先计入当前格额外的 `+1` 消耗。
3. 不可进入的障碍格、单位格不会进入可达高亮。
4. 悬停可达格时，预览路径使用 A*，并按最终路径逐格累计通行消耗；当前所在格也参与结算，但只补算当前格相对普通地板的额外代价。
5. 若目标格不在可达列表中，则不移动。
6. 若合法，则直接把主角移动到目标格。

### 4.3 当前限制

- 每回合最多移动一次
- 移动会结束可移动阶段，并进入剩余行动阶段
- 回合已结束时不能移动

### 4.4 当前未实现部分

以下移动规则目前还没有：

- 完整行动态下的多段移动决策
- 复杂地形影响

## 5. 当前结束回合规则

### 5.1 结束当前回合

可以通过以下方式触发：

- HUD 按钮 `End`
- 键盘 `T`
- 键盘 `Enter`

触发后：

- 当前玩家阶段立即结束
- 进入回合后处理
- 进入敌方空回合框架
- 自动推进到下一回合

### 5.2 当前回合状态树

当前玩家回合最小状态树为：

```text
PlayerMove
-> PlayerAction
-> TurnPost
-> EnemyTurn
-> NextTurn
```

补充：

- 玩家可以不移动，直接进入 `PlayerAction`
- 玩家移动后不会停留在 `PlayerMove`
- 玩家攻击后自动进入 `TurnPost`
- 玩家可以在 `PlayerAction` 中进入 `AttackTargeting` 或 `CardTargeting`
- `Quick` 卡牌不会立刻结束回合，而是累加 `QuickChainCount`
- 取消未提交的攻击/选牌时，若本回合尚未移动，则会回到 `PlayerMove`
- `EnemyTurn` 当前已经接入最小敌人回合执行器
- 当前默认调试敌人使用 `melee_basic` AI：若可攻击则攻击，否则向最近敌对单位靠近，否则等待

## 6. 当前普通攻击规则

### 6.1 触发方式

- 点击 `Atk` 按钮进入攻击瞄准
- 左键点击攻击范围内的敌人

### 6.2 当前判定逻辑

当前普通攻击使用最小规则：

1. 玩家进入 `AttackTargeting`
2. 用曼哈顿距离判断目标是否在攻击范围内
3. 如果目标是敌方单位或可破坏障碍物且在范围内，则造成固定伤害
4. 目标先扣护盾，再扣生命值
5. 目标 HP 归零后立即死亡并移出棋盘

### 6.3 当前已实现内容

- 攻击距离可配置
- 攻击伤害可配置
- 可破坏障碍物可被普通攻击命中
- 伤害会实时影响目标状态
- 目标死亡会移除 registry、board 占位和视图
- 敌人在 `EnemyTurn` 中会用同一套普通攻击结算攻击玩家
- 玩家 HP 归零时会触发最小失败动画并写回失败结果

### 6.4 当前未实现内容

- 视线参与的普通攻击
- 远程开枪动作接入
- 反击与命中机制

## 6.5 当前防御规则

### 6.5.1 触发方式

- 点击 `Def` 按钮

### 6.5.2 当前判定逻辑

当前防御动作使用最小规则：

1. 玩家在 `PlayerAction` 中点击 `Def`
2. 当前单位进入防御状态，持续到自己的下一个回合开始
3. 防御状态下，受到的最终伤害减少 `50%`
4. 减伤后的实际伤害向上取整
5. 防御动作会结束当前行动并进入 `TurnPost`

### 6.5.3 当前实现说明

- 当前防御效果已经做成独立接口定义，后续可继续附加：
  - 防御时获得护盾
  - 防御时触发其他天赋效果
- 当前 HUD 悬停状态会显示防御减伤百分比

## 7. 当前卡牌规则

### 7.1 当前手牌与能量

当前版本已经接入最小卡牌运行时：

- 战斗开始时抽初始手牌
- 当前能量不会每回合直接回满
- 默认每 3 回合恢复 1 点能量
- 已有抽牌堆 / 手牌 / 弃牌堆循环
- 当抽牌堆为空时，会把弃牌堆洗回抽牌堆
- 已有消耗牌堆

### 7.2 当前已接入的原型卡牌

当前原型默认牌组包括：

- `Slash`：对相邻敌人造成伤害
- `Shot`：对四方向直线上的首个敌人造成伤害
- `Meditate`：弃掉当前手牌并抽到手牌上限
- `GuardUp / Brace / QuickGuard`：提供战斗内护盾

### 7.3 当前卡牌判定逻辑

当前卡牌系统使用最小规则：

1. 玩家在 HUD 手牌区点击卡牌。
2. 若卡牌需要目标，则进入 `CardTargeting`。
3. 若卡牌是即时牌，则直接结算。
4. 结算成功后消耗能量，并把卡牌移入弃牌堆。
5. 非 `Quick` 卡牌会结束当前行动并进入 `TurnPost`。
6. `Quick` 卡牌会留在 `PlayerAction`，允许继续普通攻击或继续出牌。
7. 护盾类卡牌会直接增加当前战斗对象的护盾值。

### 7.4 当前已实现内容

- HUD 手牌卡面
- 当前能量显示
- 抽牌堆 / 弃牌堆 / 消耗牌堆查看
- 邻接目标卡牌
- 直线索敌卡牌
- 快速牌连锁计数
- 护盾牌
- 已有原型卡牌强化

### 7.5 当前未实现内容

- 数据驱动牌库资源
- 更复杂的关键词、持续效果和 buff
- 面向敌人 AI 的出牌逻辑
- 独立的冥想行动与完整流派规则

### 7.6 当前原型卡牌强化表

当前荒川强化会对每张原型卡附加一个最小强化效果：

- `cross_slash`：伤害 `+2`
- `quick_cut`：伤害 `+1`
- `line_shot`：伤害 `+2`
- `heavy_shot`：伤害 `+2`
- `battle_read`：额外抽 `1`
- `meditate`：额外抽 `1`
- `surge`：额外获得 `1` 能量
- `draw_spark`：额外抽 `1`
- `quick_plan`：额外抽 `1`
- `burning_edge`：伤害 `+2`
- `hook_shot`：伤害 `+2`
- `deep_focus`：额外抽 `1`
- `spark_charge`：额外获得 `1` 能量
- `burst_drive`：额外获得 `1` 能量
- `guard_up`：额外获得 `2` 护盾
- `brace`：额外获得 `3` 护盾
- `quick_guard`：额外获得 `2` 护盾

## 8. 当前荒川规则

### 8.1 当前荒川能量

- 荒川能量属于全局共享状态，保存在 `GlobalGameSession`
- 当前战斗 HUD 顶部会显示荒川剩余能量
- 当前原型能力每次消耗 `1` 点荒川能量
- 当前战斗内不会自动恢复；后续应由地图中的特定地点回复

### 8.2 当前荒川能力入口

- 玩家活动阶段可打开荒川轮盘
- 荒川能力不会消耗玩家的移动状态或主行动状态
- 敌方回合中无法使用荒川能力
- 点击轮盘中心可取消当前选择

### 8.3 当前已实现能力

- `造墙`：在一个空格生成不可破坏障碍物
- `强化`：强化当前手牌中的一张卡

### 8.4 当前视觉效果

- 荒川造物会触发蓝色能量闪烁与简单电磁故障感动画
- 被强化的卡牌会带蓝色遮罩并播放闪烁效果

## 9. 当前 HUD 与调试规则

当前 HUD 已被压缩成最小原型 UI。

当前固定显示内容包括：

- 当前回合阶段
- 当前能量
- 荒川能量
- 当前回能进度
- 手牌区
- 右侧操作与牌堆按钮列

当前悬停显示内容包括：

- 对象名称
- 当前生命值
- 当前护盾值

## 10. 当前护盾与障碍物规则

### 9.1 当前护盾

当前版本已经接入最小护盾规则：

- 护盾是战斗内属性，不写入 `GlobalGameSession`
- 护盾没有固定上限
- 伤害先扣护盾，再扣生命值
- 玩家和敌人都可以拥有护盾

### 9.2 当前障碍物类型

当前房间里已接入三种障碍物：

- 可破坏障碍物：有 HP，可被攻击摧毁
- 不可破坏墙体：阻挡，不可受伤
- 缓速可通过障碍物：不可受伤，但进入该格会额外 +1 移动消耗
- 若单位当前就站在缓速可通过障碍物上，则后续移动范围也会先扣掉这 `+1` 的额外消耗

## 12. 当前原型的正确理解

当前项目不是“完整战棋战斗系统”，而是已经打通以下地基的战斗原型：

- 棋盘初始化
- 统一对象模型
- 占位规则
- prefab 表现层
- HUD 状态显示
- 单回合移动测试
- 普通攻击
- 最小卡牌与能量循环
- 护盾结算
- 三类原型障碍物
- 结束回合测试

后续如果继续补系统，应当在这个原型基础上继续加：

- 攻击
- 防御
- 冥想
- 卡牌
- 能量
- 时间推进
