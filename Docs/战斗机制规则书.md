# 战斗机制规则书

## 1. 文档定位

本文件描述当前仓库里已经实现的“战斗原型规则”，不是完整成品规则书。

因此本文件只记录现在能在项目里真实验证到的规则。

## 2. 当前棋盘定义

### 2.1 棋盘

当前默认战斗棋盘来自 `BattleRoomTemplate`，默认尺寸为：

- `16 x 8`
- 单格像素尺寸 `16`

房间场景通过 `FloorLayer` 和 `MarkerLayer` 生成逻辑布局。

### 2.2 对象

当前场上对象统一为 `BoardObject`。

当前实际使用的对象类型只有三类：

- 玩家单位
- 敌人单位
- 障碍物

### 2.3 占位规则

当前规则为：

- 一个格子最多一个单位对象
- 非单位对象根据是否阻挡进入 `blocking` 或 `resident`
- 若对象 `stackable_with_unit = false`，单位不能进入该格

## 3. 当前玩家回合规则

### 3.1 当前回合状态

当前版本使用 `TurnActionState` 保存：

- `TurnIndex`
- `HasMoved`
- `HasEndedTurn`
- `QuickChainCount`

### 3.2 当前已实现动作

当前只实现了以下测试动作：

- 移动
- 结束当前回合
- 进入下一回合
- 调整玩家移动力

## 4. 当前移动规则

### 4.1 触发方式

- 鼠标左键点击棋盘格

### 4.2 当前判定逻辑

当前是原型版判定，不是完整战术寻路：

1. 先取主角当前格。
2. 用曼哈顿距离生成可达格列表。
3. 若目标格不在可达列表中，则不移动。
4. 若目标格违反占位规则，则不移动。
5. 若合法，则直接把主角移动到目标格。

### 4.3 当前限制

- 每回合最多移动一次
- 移动不会结束回合
- 回合已结束时不能移动

### 4.4 当前未实现部分

以下移动规则目前还没有：

- A* 寻路
- 按路径逐格累计地形消耗
- 路径中途阻挡截断
- 复杂地形影响

## 5. 当前结束回合规则

### 5.1 结束当前回合

可以通过以下方式触发：

- HUD 按钮 `End`
- 键盘 `T`
- 键盘 `Enter`

触发后：

- `HasEndedTurn = true`
- 本回合不再允许移动

### 5.2 进入下一回合

当当前回合已经结束后，再次触发结束回合入口，会进入下一回合：

- HUD 按钮会显示为 `Next`
- `TurnIndex` 增加
- `HasMoved` 重置
- `HasEndedTurn` 重置

## 6. 当前 HUD 与调试规则

当前 HUD 是底部状态条，不是完整战斗 UI。

当前显示内容包括：

- 玩家 HP
- 玩家移动力
- 玩家所在格
- 当前回合号
- 本回合是否已移动
- 本回合是否已结束
- 对象数量摘要
- 当前动画名摘要

当前按钮包括：

- `MV-`
- `MV+`
- `End` / `Next`

## 7. 当前尚未实现的规则

以下机制尚未进入当前规则书的“已实现部分”：

- 普通攻击
- 防御
- 冥想
- 卡牌使用
- 能量消耗与回复
- 视线判定
- targeting 规则
- 时间推进
- 敌人 AI 回合
- 战斗结算

## 8. 当前原型的正确理解

当前项目不是“完整战棋战斗系统”，而是已经打通以下地基的战斗原型：

- 棋盘初始化
- 统一对象模型
- 占位规则
- prefab 表现层
- HUD 状态显示
- 单回合移动测试
- 结束回合测试

后续如果继续补系统，应当在这个原型基础上继续加：

- 攻击
- 防御
- 冥想
- 卡牌
- 能量
- 时间推进
