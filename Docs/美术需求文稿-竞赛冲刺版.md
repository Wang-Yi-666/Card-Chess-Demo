# 美术需求文稿（竞赛冲刺版）

## 1. 文档定位

这不是长期商业项目的完整美术圣经，而是一份直接给当前美术同学执行的冲刺需求文稿。

项目现实条件：

- 开发周期不足半个月
- 美术仅 1 人
- 作品目标是极限开发竞赛提交版本

因此，美术策略必须是：

- 优先统一风格
- 优先保证可读性
- 优先保证可落地
- 坚决控制资源规模

不追求“大而全”，追求“少而准、能撑起核心体验”。

## 2. 总体美术方向

### 2.1 视觉类型

主要采用：

- 2D 复古 RPG 地图
- 像素 sprite 角色
- 16x16 像素 tilemap

参考方向：

- GBA 时代《火焰纹章》的战场阅读性
- GBA / NDS 时代《宝可梦》的地图探索可读性

不是要完全复刻它们，而是借用它们的优点：

- 小尺寸像素图也能读清角色和地图
- 地图层级简单明确
- 单位、地块、交互物一眼可分

### 2.2 当前最重要的视觉目标

这次竞赛版本，美术最核心的任务不是做很多内容，而是让玩家第一眼就能看懂：

- 角色是谁
- 敌人是谁
- 地图哪里能走
- 哪些东西能交互
- 战斗场地有哪些地形差异
- 卡牌和按钮分别代表什么

## 3. 风格关键词

建议整体风格关键词：

- 像素
- 复古
- 冷调科幻
- 边境废土
- 工业残骸
- 星际牛仔
- 幽蓝异常能量

### 3.1 基础颜色方向

地图与场景基底：

- 冷灰
- 铁锈棕
- 暗蓝灰
- 脏白

主角识别色：

- 暗红 / 红色方巾

荒川 / 异能识别色：

- 幽蓝
- 蓝白电弧

危险 / 敌人 / 警示色：

- 暖红
- 橙黄

## 4. 本次版本的美术优先级

按优先级从高到低：

### P0 必做

- 主角像素角色
- 荒川相关最小视觉符号
- 基础敌人像素角色
- 地图 tilemap 基础地块
- 战斗房间 tilemap 基础地块
- 三类障碍物区分
- 基础 UI 风格统一
- 卡牌基础视觉统一

### P1 应做

- 主角多帧待机 / 移动感
- 敌人多轮廓差异
- 幽灵船 / 星港基础风格元素
- 荒川能力特效基础版
- 牌堆查看和构筑界面的统一卡面风格

### P2 可选

- 更复杂场景装饰
- 更完整角色立绘
- 更多敌人细分职业
- 更复杂地图动画和环境特效

## 5. 具体资源需求

## 5.1 主角

### 核心定位

主角是“失忆的星际牛仔”，视觉上必须一眼有辨识度。

### 像素角色需求

建议规格：

- 基础角色占用：16x16 或 16x24 的视觉体量
- 在当前项目中允许通过缩放放大显示

建议最少方向帧：

- 上
- 下
- 左右

建议最少状态：

- 待机
- 移动
- 受击 / 行动可选做简化

### 必须保留的识别元素

- 红色方巾
- 长外套或披挂感
- 不是军人，而是漂泊者

## 5.2 荒川

这次版本不要求做完整角色立绘优先，但至少要有：

- 一个稳定的视觉识别符号
- 能体现“幽蓝、故障、AI异常感”

建议资源：

- 荒川按钮 / 图标
- 荒川能力相关图标
- 一个最小投影头像或象征性半身图

## 5.3 敌人

### 目标

本次版本不需要做很多种敌人，但需要做到：

- 和主角明显区分
- 一眼能看出是敌方单位
- 至少有 1 到 2 种轮廓变化

### 建议资源

- 基础敌人士兵
- 稍强一点的敌人变体

颜色建议：

- 偏暗黄、暗棕、脏红
- 避免和主角的红方巾、荒川的幽蓝冲突

## 6. 地图与 tilemap 需求

## 6.1 地图风格

地图主要采用复古 RPG 式俯视 2D tilemap。

主要目标：

- 可读
- 易拼接
- 少而精

### 地图 tile 尺寸

- 统一 16x16

### 地图资源建议分层

- 地板
- 墙体 / 边界
- 装饰
- 交互物

## 6.2 当前最需要的地图 tile 组

### 基础地板

- 金属地板
- 脏旧舱内地板
- 星港平台地板

### 墙体与结构

- 舱壁
- 门框
- 管线 / 栏杆 / 隔断

### 装饰

- 箱体
- 终端
- 警示灯
- 维修零件

### 交互物

- 宝箱 / 箱体
- 治疗站(根据世界观文本来说是类似充电站一样的东西)
- 门
- 遇敌点 / 敌人站位点

## 6.3 战斗房间 tilemap 需求

当前战斗场地不是大地图，而是小型战斗房间池。

这部分美术必须优先考虑“玩法阅读性”。

### 本次战斗房间必须做清的地块

- 普通可行走地板
- 不可破坏墙体
- 可破坏障碍物
- 缓速可通过障碍物

玩家必须一眼区分这三类障碍物。

### 为什么这部分重要

因为房间池和地形池是当前项目制造随机感的关键来源之一。

同一类敌人，放在不同房间布局里，会形成不同战术题面。

所以这里不是简单换背景，而是玩法的一部分。

## 7. UI 美术需求

## 7.1 总体要求

竞赛版本 UI 不追求华丽，追求：

- 一致
- 清晰
- 复古终端感

建议方向：

- 像素边框
- 少量工业面板感
- 少量冷色高亮

不要做成：

- 现代手游大渐变
- 玻璃拟态
- 重霓虹炫光

## 7.2 当前最关键的 UI 区域

- 战斗 HUD 顶栏
- 右侧动作按钮列
- 底部手牌区
- 牌堆查看弹窗
- 局外构筑界面

## 7.3 卡牌视觉需求

当前卡牌已经是一个核心信息载体，要求：

- 一眼分清攻击 / 技能 / 治疗
- 费用清楚
- 标题清楚
- 类型与关键特性清楚

### 这次版本的建议

先不要给每张卡都做高成本完整插画。

更合理的是：

- 统一卡框
- 统一类型色
- 用小型符号插画或简化卡图
- 少数关键卡做重点图区分

## 8. 特效需求

## 8.1 必须保留的特效核心

- 荒川能力的幽蓝异常感
- 攻击命中反馈
- 护盾获得反馈
- 治疗反馈

## 8.2 特效优先级

### P0

- 荒川能力基础蓝光 / 电弧
- 攻击命中
- 治疗数值 / 护盾数值反馈

### P1

- 角色行动残影
- 房间环境光变化
- 牌面强化脉冲

## 9. 实际制作建议

## 9.1 单人美术的工作策略

这次最重要的不是“画很多”，而是“先建立规则，再批量复制”。

建议顺序：

1. 先做一套基础 tile 风格
2. 先做主角和基础敌人的统一比例
3. 先做一套统一卡牌框架
4. 再做少量关键特效

也就是说：

- 先做系统
- 再做内容

## 9.2 不建议做的内容

本次版本不建议把时间投入到：

- 复杂大立绘
- 大量环境装饰细节
- 多套主角服装
- 复杂角色表情系统
- 大量高帧动画

这些都很贵，但对竞赛版本的直接价值不如“可读性和统一性”。

## 10. 本次美术交付清单建议

如果按最小可交付版本列清单，建议是：

### 角色

- 主角像素角色 1 套
- 敌人像素角色 1 到 2 套
- 荒川图标 / 头像 1 套

### 地图

- 基础飞船 / 星港 tile 1 套
- 战斗房间基础 tile 1 套
- 三类障碍物表现 1 套

### UI

- 卡牌视觉框架 1 套
- HUD 风格统一 1 套
- 构筑界面风格统一 1 套

### 特效

- 攻击 / 治疗 / 护盾 / 荒川能力基础版

## 11. 结论

本次竞赛版本的美术策略应当非常明确：

- 采用 16x16 像素 tilemap
- 采用 GBA 时代复古 RPG / 战棋式阅读逻辑
- 优先统一风格和可读性
- 优先完成主角、敌人、战斗地形、卡牌和 HUD
- 房间地形池要服务玩法随机感，不只是做背景

如果这条路线执行稳定，那么即使资源量不大，作品也会有明确的完成度和辨识度。
