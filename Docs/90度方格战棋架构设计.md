# 90度方格战棋架构设计

## 1. 文档定位

本文件描述当前仓库里已经落地的战斗原型架构，而不是未来完整战斗系统的理想蓝图。

当前战斗原型的特点是：

- 独立运行
- 以房间模板初始化棋盘
- 以 `BoardObject` 统一管理玩家、敌人、障碍物
- 已接通表现层和 HUD
- 只实现了“移动测试 + 回合测试”

## 2. 当前核心入口

当前战斗入口是：

- 场景：`Scene/Battle/Battle.tscn`
- 控制器：`Scripts/Battle/BattleSceneController.cs`

`BattleSceneController` 是当前战斗原型的总调度器。

当前默认主场景里的关键节点还包括：

- `Camera2D`
- `RoomContainer`
- `RoomContainer/BoardOverlay`
- `RoomContainer/PieceRoot`
- `BattleHud`

## 3. 当前模块拆分

### 3.1 Shared

- `GlobalGameSession`

当前只负责玩家基础运行时状态，不是完整战斗上下文。

### 3.2 Data

- `RoomLayoutDefinition`
- `BoardObjectSpawnDefinition`

负责表达房间布局输出和对象出生定义。

### 3.3 Board

- `BoardState`
- `BoardCellState`
- `BoardObject`
- `BoardObjectRegistry`
- `BoardInitializer`
- `BoardQueryService`
- `BoardTopology`
- `OccupancyRules`

这是当前逻辑层地基。

### 3.4 Rooms

- `BattleRoomTemplate`
- `BattleRoomPoolDefinition`
- `BattleRoomPoolEntry`
- `BattleRoomTileSetFactory`

负责把 TileMap 房间模板转换成可初始化的逻辑布局。

### 3.5 State

- `BattleObjectState`
- `BattleObjectStateManager`

负责把逻辑对象状态转成更适合 HUD 和 prefab 使用的运行时状态。

### 3.6 Actors

- `BattleActorTokenBase`
- `BattlePlayerToken`
- `BattleEnemyToken`
- `BattleObstacleToken`

这组类是仓库中仍然保留的辅助 token 脚本，但当前默认战斗 prefab 没有直接引用它们。

### 3.7 Presentation

- `BattlePrefabEntry`
- `BattlePrefabLibrary`
- `BattleAnimatedViewBase`
- `BattlePlayerView`
- `BattleEnemyView`
- `BattleObstacleView`
- `BattlePieceViewManager`

负责 prefab 视图和基础动画。

### 3.8 Visual

- `BattleBoardOverlay`

负责当前的高亮、悬停和路径预览。

### 3.9 UI

- `BattleHudController`

负责底部状态条、回合按钮和移动力调试按钮。

### 3.10 Turn

- `TurnActionState`

当前只保存玩家回合内的最小状态，没有独立的 `TurnFlowController`。

## 4. 当前初始化流程

1. `BattleSceneController` 创建 `BoardState`、`BoardObjectRegistry`、`BoardQueryService`、`TurnActionState`
2. 从房间池或导出场景中选出一个 `BattleRoomTemplate`
3. `BattleRoomTemplate.BuildLayoutDefinition()` 生成 `RoomLayoutDefinition`
4. `BoardInitializer.InitializeFromLayout(layout)` 初始化棋盘
5. `BattleObjectStateManager` 同步对象状态
6. `BattlePieceViewManager` 创建玩家、敌人、障碍物 prefab
7. `BattleBoardOverlay` 绑定当前房间
8. `BattleHudController` 绑定状态与回合数据
9. `ConfigureCameraForBattle()` 根据棋盘尺寸和 HUD 预留空间重设主摄像机
10. 控制器订阅 `GlobalGameSession` 与 HUD 发出的调试/回合信号

## 5. 当前 board 模型

### 5.1 当前格子结构

每个格子目前包含：

- `Cell`
- `TerrainId`
- `BaseMoveCost`
- `UnitObjectId`
- `ResidentObjectIds`
- `BlockingObjectIds`
- `Tags`

### 5.2 当前对象结构

当前 `BoardObject` 已支持：

- `object_id`
- `definition_id`
- `object_type`
- `cell`
- `faction`
- `tags`
- `max_hp`
- `current_hp`
- `blocks_movement`
- `blocks_line_of_sight`
- `stackable_with_unit`
- `move_cost_modifier`
- `state_payload`

### 5.3 当前占位规则

当前版本已经落地：

- 一个格子最多一个单位对象
- 非单位对象根据阻挡性进入 `blocking` 或 `resident`
- 不允许与单位同格的对象会阻止单位进入

## 6. 当前房间系统

当前不是通过独立布局文件直接喂给控制器，而是：

- 房间场景里维护 `FloorLayer` 和 `MarkerLayer`
- `MarkerLayer` 用 tile alternative 标记玩家、敌人、障碍物
- `BattleRoomTemplate` 运行时把这些标记转换成 `RoomLayoutDefinition`

这意味着当前房间内容的主要编辑入口仍然是 Godot 场景，而不是纯数据表。

## 7. 当前移动系统

当前移动逻辑是“原型版”，不是完整战术寻路系统。

已经实现：

- 按玩家移动力计算可达格
- 左键点击目标格移动
- 每回合最多移动一次
- HUD 和 overlay 同步反映移动状态

当前尚未实现：

- A* 寻路
- 按路径逐格累计通行消耗
- 路径阻挡截断
- 视线
- 攻击目标选择

当前可达判断是简化的曼哈顿距离判断，主要用于验证棋盘状态、表现层和回合状态能否接通。

## 8. 当前回合系统

当前回合层只包含一个最小状态对象：

```text
TurnActionState
- TurnIndex
- HasMoved
- HasEndedTurn
- QuickChainCount
- CanMove
- CanAct
```

当前行为：

- 玩家每回合只能移动一次
- 移动不会结束回合
- 可以显式结束回合
- 再次触发会进入下一回合

当前还没有：

- 主行动系统
- 敌方回合
- 时间推进
- 动作队列

## 9. 当前表现层架构

当前运行时显示采用两层分离：

1. 逻辑层：`BoardObjectRegistry`
2. 表现层：`BattlePieceViewManager` + prefab scenes

对应关系为：

- `battle_player` -> `BattlePlayerToken.tscn`
- `battle_enemy` -> `BattleEnemyToken.tscn`
- `battle_obstacle` -> `BattleObstacleToken.tscn`

HUD 不再直接读取场景节点，而是读取 `BattleObjectStateManager`。

另外还有两组真实存在但默认未接入主链路的辅助内容：

- `BattleBoardDebugView`：可直接绘制 `BoardState` 调试棋盘，但 `Battle.tscn` 当前未挂接
- `Scripts/Battle/Actors/*`：提供另一套自绘 token 脚本，当前 `DefaultBattlePrefabLibrary.tres` 没有引用这组类

## 10. 当前未纳入架构的系统

以下内容过去常被写进设计文档，但在当前代码中尚未落地：

- 卡牌系统
- 能量系统
- 普通攻击
- 防御
- 冥想
- `TargetingService`
- `TurnFlowController`
- 敌人 AI
- 时间推进
- `BattleRequest` / `BattleResult`

## 11. 当前架构结论

如果只看当前仓库，最准确的描述是：

这是一个“房间模板驱动的战棋原型”，已经打通了：

- object 建模
- 占位规则
- 房间初始化
- prefab 表现
- HUD 状态显示
- 单回合移动测试

但尚未进入完整战斗规则层。
