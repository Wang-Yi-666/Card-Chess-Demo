# 90度方格战棋架构设计

## 1. 文档定位

本文件描述当前仓库里已经落地的战斗原型架构，而不是未来完整战斗系统的理想蓝图。

当前战斗原型的特点是：

- 独立运行
- 以房间模板初始化棋盘
- 以 `BoardObject` 统一管理玩家、敌人、障碍物
- 已接通表现层和 HUD
- 只实现了“移动测试 + 回合测试”

## 2. 当前核心入口

当前战斗入口是：

- 场景：`Scene/Battle/Battle.tscn`
- 控制器：`Scripts/Battle/BattleSceneController.cs`

`BattleSceneController` 是当前战斗原型的总调度器。

当前默认主场景里的关键节点还包括：

- `Camera2D`
- `RoomContainer`
- `RoomContainer/BoardOverlay`
- `RoomContainer/PieceRoot`
- `BattleHud`

## 3. 当前模块拆分

### 3.1 Shared

- `GlobalGameSession`

当前只负责玩家基础运行时状态，不是完整战斗上下文。

### 3.2 Data

- `RoomLayoutDefinition`
- `BoardObjectSpawnDefinition`

负责表达房间布局输出和对象出生定义。

### 3.3 Encounters

- `BattleEncounterProfile`
- `BattleEncounterLibrary`

负责把地图侧未来传入的 `EncounterId` 解析成敌人定义和房间筛选条件。

### 3.4 Board

- `BoardState`
- `BoardCellState`
- `BoardObject`
- `BoardObjectRegistry`
- `BoardInitializer`
- `BoardQueryService`
- `BoardTopology`
- `BoardPathfinder`
- `BoardTargetingService`
- `OccupancyRules`

这是当前逻辑层地基。

### 3.5 Rooms

- `BattleRoomTemplate`
- `BattleRoomPoolDefinition`
- `BattleRoomPoolEntry`
- `BattleRoomTileSetFactory`

负责把 TileMap 房间模板转换成可初始化的逻辑布局。

### 3.6 State

- `BattleObjectState`
- `BattleObjectStateManager`

负责把逻辑对象状态转成更适合 HUD 和 prefab 使用的运行时状态。

### 3.7 Actions

- `BattleActionService`

负责把移动、普通攻击这类通用战斗动作从控制器中抽离，供玩家输入和敌人 AI 共用。

### 3.8 AI

- `IEnemyAiStrategy`
- `EnemyAiRegistry`
- `EnemyTurnResolver`

负责按 `AiId` 选择敌人策略，并在 `EnemyTurn` 中顺序执行最小敌方行为。

### 3.9 Actors

- `BattleActorTokenBase`
- `BattlePlayerToken`
- `BattleEnemyToken`
- `BattleObstacleToken`

这组类是仓库中仍然保留的辅助 token 脚本，但当前默认战斗 prefab 没有直接引用它们。

### 3.10 Presentation

- `BattlePrefabEntry`
- `BattlePrefabLibrary`
- `BattleAnimatedViewBase`
- `BattlePlayerView`
- `BattleEnemyView`
- `BattleObstacleView`
- `BattlePieceViewManager`

负责 prefab 视图和基础动画。

### 3.11 Visual

- `BattleBoardOverlay`

负责当前的高亮、悬停和路径预览。

### 3.12 UI

- `BattleHudController`

当前只保留回合按钮和悬停状态卡片。

### 3.13 Turn

- `TurnActionState`

当前保存战斗回合的最小 phase / input 状态；敌方执行细节已经移到 `EnemyTurnResolver`。

## 4. 当前初始化流程

1. `BattleSceneController` 创建 `BoardState`、`BoardObjectRegistry`、`BoardQueryService`、`TurnActionState`
2. 控制器从 `BattleEncounterLibrary` 解析 `EncounterId`
3. 从房间池或导出场景中选出一个 `BattleRoomTemplate`
4. `BattleRoomTemplate.BuildLayoutDefinition(enemyDefinitionId)` 生成 `RoomLayoutDefinition`
5. `BoardInitializer.InitializeFromLayout(layout)` 初始化棋盘
6. 创建 `BoardPathfinder` 和 `BoardTargetingService`
7. `BattleObjectStateManager` 同步对象状态
8. `BattlePieceViewManager` 创建玩家、敌人、障碍物 prefab
9. 创建 `BattleActionService` 和 `EnemyTurnResolver`
10. `BattleBoardOverlay` 绑定当前房间
11. `BattleHudController` 绑定回合按钮与悬停卡片
12. `ConfigureCameraForBattle()` 根据棋盘尺寸和 HUD 预留空间重设主摄像机

## 5. 当前 board 模型

### 5.1 当前格子结构

每个格子目前包含：

- `Cell`
- `TerrainId`
- `BaseMoveCost`
- `UnitObjectId`
- `ResidentObjectIds`
- `BlockingObjectIds`
- `Tags`

### 5.2 当前对象结构

当前 `BoardObject` 已支持：

- `object_id`
- `definition_id`
- `object_type`
- `cell`
- `faction`
- `tags`
- `max_hp`
- `current_hp`
- `blocks_movement`
- `blocks_line_of_sight`
- `stackable_with_unit`
- `move_cost_modifier`
- `state_payload`

### 5.3 当前占位规则

当前版本已经落地：

- 一个格子最多一个单位对象
- 非单位对象根据阻挡性进入 `blocking` 或 `resident`
- 不允许与单位同格的对象会阻止单位进入

## 6. 当前房间系统

当前不是通过独立布局文件直接喂给控制器，而是：

- 房间场景里维护 `FloorLayer` 和 `MarkerLayer`
- `MarkerLayer` 用 tile alternative 标记玩家、敌人、障碍物
- `BattleRoomTemplate` 运行时把这些标记转换成 `RoomLayoutDefinition`

这意味着当前房间内容的主要编辑入口仍然是 Godot 场景，而不是纯数据表。

## 7. 当前移动系统

当前移动逻辑是“原型版”，不是完整战术寻路系统。

已经实现：

- `BoardPathfinder` 计算可达格
- `BoardPathfinder` 提供 A* 路径预览
- 可达格和 A* 预览路径都按路径逐格累计通行消耗
- 若当前脚下是慢速可通过障碍格，会先补算当前格相对普通地板的额外消耗
- 左键点击目标格移动
- 每回合最多移动一次
- 障碍格和不可进入格不会进入可达高亮
- 移动完成后自动退出移动阶段

当前尚未实现：

- 面向行动系统的完整移动状态树
- 攻击动作接入

当前可达区域已经不再是纯曼哈顿高亮，而是经过棋盘占位规则过滤；预览路径已切到 A* 工具层。

## 8. 当前回合系统

当前回合层现在是一个最小状态树：

```text
TurnActionState
- TurnIndex
- Phase
  - PlayerMove
  - PlayerAction
  - TurnPost
  - EnemyTurn
- InputMode
  - None
  - AttackTargeting
- HasMoved
- HasActed
- HasEndedTurn
- QuickChainCount
- CanMove
- CanEnterAttackTargeting
- IsAttackTargeting
```

当前行为：

- 玩家可以跳过移动
- 玩家每回合最多移动一次
- 移动后自动进入剩余行动阶段
- 玩家可以进入普通攻击瞄准状态
- 普通攻击完成后进入回合后处理
- `EnemyTurn` 会调用 `EnemyTurnResolver`
- 当前默认调试敌人使用 `melee_basic`，每个敌人一回合只执行一个最小动作：
  - 若目标在普攻范围内，则普通攻击
  - 否则向最近敌对单位靠近
  - 若无法推进，则等待

当前还没有：

- 完整主行动系统
- 敌方 AI 逻辑
- 时间推进
- 动作队列

## 8.1 当前普通攻击系统

当前已实现最小普通攻击：

- 通过 `Atk` 按钮进入攻击瞄准状态
- 用曼哈顿距离判定攻击距离
- 攻击距离与伤害值来自对象运行时状态
- 攻击会实时减少目标 HP
- 目标 HP 归零后会从棋盘、registry 和视图层移除

当前尚未实现：

- 视线参与的普通攻击判定
- 攻击动画时序与结算分离
- 防御 / 反击 / 命中判定
- 敌方普通攻击

## 9. 当前表现层架构

当前运行时显示采用两层分离：

1. 逻辑层：`BoardObjectRegistry`
2. 表现层：`BattlePieceViewManager` + prefab scenes

对应关系为：

- `battle_player` -> `BattlePlayerToken.tscn`
- `battle_enemy` -> `BattleEnemyToken.tscn`
- `battle_obstacle` -> `BattleObstacleToken.tscn`

HUD 不再直接读取场景节点，而是读取 `BattleObjectStateManager`。

另外还有两组真实存在但默认未接入主链路的辅助内容：

- `BattleBoardDebugView`：可直接绘制 `BoardState` 调试棋盘，但 `Battle.tscn` 当前未挂接
- `Scripts/Battle/Actors/*`：提供另一套自绘 token 脚本，当前 `DefaultBattlePrefabLibrary.tres` 没有引用这组类

## 10. 当前未纳入架构的系统

以下内容过去常被写进设计文档，但在当前代码中尚未落地：

- 卡牌系统
- 能量系统
- 防御
- 冥想
- `TurnFlowController`
- 敌人 AI
- 时间推进
- `BattleRequest` / `BattleResult`

补充说明：

- `BoardTargetingService` 已经存在，但目前只提供工具层的直线索敌查询
- 它还没有正式接入攻击、卡牌或敌人行动

## 11. 当前架构结论

如果只看当前仓库，最准确的描述是：

这是一个“房间模板驱动的战棋原型”，已经打通了：

- object 建模
- 占位规则
- 房间初始化
- prefab 表现
- HUD 状态显示
- 单回合移动测试

但尚未进入完整战斗规则层。
